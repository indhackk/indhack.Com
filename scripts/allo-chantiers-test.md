## 3. Noms de Domaine Proposés

### NDD #1 : `etancheite-haguenau.fr`

| Aspect | Analyse |
|--------|---------|
| **Pourquoi ce choix** | EMD (Exact Match Domain) local fort - Keyword principal + ville directement dans le domaine = signal géo clair pour Google. Mémorisable, professionnel. |
| **Risques SEO** | EMD = léger risque de filtre Google si sur-optimisation ou contenu faible. Nécessite contenu de qualité pour éviter le spam score. |
| **Compatibilité scale** | **Moyenne** - Limitation si expansion multi-villes (perçu comme mono-ville). Nécessite création de nouveaux domaines pour autres marchés OU évolution vers structure sous-domaines. |

### NDD #2 : `alsace-etancheite-toiture.fr`

| Aspect | Analyse |

## 4. 5 Règles Anti-Duplicate / Anti-Spam pour Déployer 20 Villes

### Règle 1 : Contenu Unique à 60% Minimum par Page Ville

> Chaque page ville doit avoir au minimum **60% de contenu unique** :
> - Introduction localisée (150 mots min) avec contexte géographique
> - Zones d'intervention spécifiques (communes, quartiers)
> - Témoignages clients locaux (prénom + ville)
> - Photos de chantiers de la zone si disponibles
> - Landmarks locaux mentionnés naturellement
>
> Le template commun (process, garanties, FAQ) ne doit pas dépasser 40%.

### Règle 2 : Données Structurées LocalBusiness Distinctes

> Chaque page ville = **1 schema LocalBusiness avec données uniques** :
> - Adresse/téléphone/areaServed différent
> - Coordonnées GPS spécifiques à la ville
> - Jamais le même NAP (Name, Address, Phone) sur 2 pages villes
> - Service area adapté aux communes couvertes depuis cette ville

### Règle 3 : Maillage Interne Hub & Spoke

> Page `/zones-intervention/` = **hub central** qui liste toutes les villes.
> Chaque page ville linke vers :
> - Le hub zones (obligatoire)
> - 2-3 villes voisines max (maillage horizontal limité)
> - La home page (money page)
>
> **Éviter le maillage "tous vers tous"** = signal spam pour Google.

### Règle 4 : Canonical Self-Referencing + Index Follow

> - Home = canonical self-referencing
> - Pages villes = index, follow avec canonical vers elle-même
> - **Pas de canonical croisé entre pages villes**
> - Si contenu temporairement trop similaire = noindex jusqu'à enrichissement

### Règle 5 : 1 GMB = 1 Ville = 1 Adresse Réelle

> Pour chaque ville déployée avec volume suffisant :
> - Ouvrir un GMB distinct avec **adresse physique vérifiable**
> - Pas de GMB "virtuel" multi-zones sans présence réelle
> - Cohérence NAP obligatoire : site ↔ GMB ↔ Citations ↔ Annuaires
> - Alternative : SAB (Service Area Business) si pas d'adresse locale, mais moins performant pour Local Pack
